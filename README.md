### Practice 11-12

### API-шар

Логіка роботи з сервером винесена в окремий файл js/api.js. Реалізовані функції для всіх основних HTTP-операцій:

- отримання списку записів
- отримання запису за id
- створення нового запису
- оновлення запису
- видалення запису

Це дозволяє уникнути дублювання fetch-запитів у різних частинах застосунку.

### Отримання та відображення даних

Дані завантажуються через GET-запит до /items і динамічно відображаються у вигляді списку або таблиці. Розмітка генерується на основі отриманих даних.

### Стани інтерфейсу

Реалізовано обробку основних станів:

- loading — відображається під час завантаження
- error — у випадку помилки запиту
- empty — якщо список порожній

### Створення записів

Додано форму для створення нового елемента. Дані збираються через FormData і відправляються на сервер методом POST. Після успішного створення список оновлюється.

### Валідація

Перед відправленням форми перевіряються:

- обов’язкові поля
- коректність числових значень

При помилках користувачу відображаються повідомлення.

### Редагування записів

Кожен запис має кнопку редагування. Дані підставляються у форму, після чого можна змінити значення і відправити PATCH або PUT запит. Інтерфейс оновлюється після відповіді сервера.

### Видалення записів

Реалізовано видалення через DELETE-запит з підтвердженням дії. Після видалення елемент зникає зі списку без перезавантаження сторінки.

### Пошук, фільтрація, сортування, пагінація

Реалізовано роботу з query-параметрами:

- пошук (q)
- фільтрація за категорією
- сортування (\_sort, \_order)
- пагінація (\_page, \_limit)

При зміні параметрів список повторно завантажується.

### Синхронізація інтерфейсу

Після кожної CRUD-операції інтерфейс оновлюється відповідно до стану сервера. Виключено дублювання записів та відображення застарілих даних.

## Додаткові можливості

- збереження стану фільтрів у URL
- окрема сторінка деталей запису
- оптимістичне оновлення інтерфейсу
- кастомне модальне вікно підтвердження видалення

## Запуск проєкту

1. Встановити json-server:

```
npm install -g json-server
```

2. Запустити сервер:

```
json-server --watch db.json
```

3. Відкрити index.html у браузері за допомогою Live-server

API буде доступне за адресою:

```
http://localhost:3000/items
```
